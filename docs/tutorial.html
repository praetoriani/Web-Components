How to create Web Components 🤨?

Okay, so here we are. You want to know how you can develop your own reusable web components?
Then let's start straight away.

First of all, a small checklist of everything you need to complete this tutorial.
  • Basic knowledge of HTML, CSS and Javascript
  • Your favorite editor (I use VS Code)
To test our web component, we only need a modern browser (like Google Chrome or Microsoft Edge). Modern browsers are able to interpret pure HTML, CSS and Javascript. So there is no need for an extra tool to test our web component.

However, if you want to test your web component with a local web server, I can recommend the following options: I'm using NodeJS and the NPM module http-server. Both tools are quickly installed and easy to set up. An alternative to this would be ngrok.
  • Download NodeJS
  • http-server (npm module)
  • Download ngrok
Each web component consists of three parts. An HTML part, a CSS part and a Javascript part. For the sake of simplicity, I first create an HTML file, a CSS file and a JS file for each new web component.

In the end, the HTML file is only used to test our web component. Because the actual web component is dynamically generated by Javascript (but more on that later). In the CSS file we will write the code that is responsible for styling our web component.

Everything clear so far? Then let's start with a small example.


Web-Component: Info-Banner

In order not to make things too complicated at the beginning, we will create a simple web component here. An information banner. This info-banner component will have a few properties that we can dynamically change. In the end our component will look like this:

Code Example
copied
<info-banner
   text="Hello, Web Components!"
   position="bottom"
   bgcolor="#004CDA"
   fontcolor="#FEFEFE">
</info-banner>

But it will still take a while until then. From here I will explain to you step by step my process for developing a web component like this one. And in this case I first create the folder info-banner.
In this folder I create the following three files:

  • info-banner.html
  • info-banner.css
  • info-banner.js

I then open these three files in my code editor. The HTML file first receives the base code. And so that we don't forget, we'll also include our CSS file and the JS file here. So your code should look like this:
(the <html lang="de"> might be different in your code)

info-banner.html
copied
<!DOCTYPE html>
<html lang="de">
<head>
   <meta charset="UTF-8">
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   <title>Web-Component: info-banner</title>
   <script src="info-banner.js"></script>
   <link rel="stylesheet" href="info-banner.css">
</head>
<body>

</body>
</html>

From here we could start writing our code for the web component in the JS file. To make it clear how such a web component is structured, I will first create our web component in the HTML file in the next step. In principle, this intermediate step is not necessary, but it can help, especially with more complex components.

If you are a beginner when it comes to HTML, CSS and Javascript, I recommend that you first create your component as HTML code and then implement it later in the Javascript code.

Our info banner will be a DIV Layer that we can dynamically display at the top or bottom of the page. When creating the web component, it is important that we strictly separate the code. So don't use inline CSS or inline Javascript code.

So we now add the DIV Layer to our HTML file. We place the code between the body tags.

info-banner.html
copied
<body>
   <div class="BannerDesign Roboto400">
      <p>
      <!-- placeholder for the text-->
      </p>
   </div>
</body>

Since we write the CSS code for our DIV Layer in our info-banner.css, no further changes are necessary in the HTML file. So we now open our info-banner.css and define the appearance of our DIV Layer there. I'm going to use Google Fonts (Roboto Sans) in this example for a nice font. Our DIV Layer will be 800px wide and 40px high and (depending on your needs) will be horizontally centered and aligned at the top or bottom. Let's give it a nice, blue background color with white font. Add a dark colored frame with rounded corners and we have our first layout. Our CSS file should now look like this:

info-banner.css
copied
/* Import Roboto from fonts.google.com */
@import url('https://fonts.googleapis.com/css2?family=Roboto:wght@100;300;400;500;700&display=swap');

.Roboto400 {
   font-family: "Roboto", sans-serif;
   font-weight: 400;
   font-style: normal;
}

.BannerDesign {
   width: 800px;
   height: 40px;
   position: fixed;
   left: 50%;
   transform: translateX(-50%);
   margin: 0px;
   padding: 6px;
   border: 1px solid rgba(51,51,51,0.8);
   border-radius: 8px;
   background-color: #00509C;
   display: table;
   font-size: 16px;
   color: #EFEFEF;
}

.BannerDesign p {
   display: table-cell;
   vertical-align: middle;
   text-align: center;
}

With the code from the HTML and CSS files, we have already created a working template. There is no Javascript code yet. We have defined the DIV layer in our HTML page and have written the CSS code for the DIV layer in our CSS file.

Maybe a few more words about the CSS code:
In the current code, the DIV layer is currently horizontally centered. The text in the DIV layer is also centered horizontally and vertically. We will later dynamically adjust the alignment (top or bottom) using our Javascript code. The code is really only used to create the template. The CSS code is also later manipulated via the Javascript code.

OK. Then we are now ready to write our Javascript code. One more thing in advance. The purpose behind a web component is to create a self-contained and isolated HTML element that is independent of the rest of the code and can still interact with it. To achieve this, we will create a class in our Javascript code that will render our HTML element in a virtual Shadow DOM. And the HTML page ultimately renders this Shadow DOM for us as an isolated and independent element.

Yes I know. It all sounds a bit complicated at first. But once we have the Javascript code ready, some things will be much clearer than they are now. So let's create our Javascript file. In the first step we just define the class and the name of our web component.

info-banner.js
copied
class InfoBanner extends HTMLElement {

}
window.customElements.define('info-banner', InfoBanner);

The Javascript code above is the basic framework for our web component. We will now define everything that has to do with our web component within this class. So in the next step we will define a few functions in our class. For our web component we definitely need the following functionalities:

We need a constructor that allows us to access the Shadow DOM. We also need a function that creates our component and renders it within the Shadow DOM. To make our component dynamic, we need another function that can update the component. But that's still not everything. Our component should have certain attributes that we can set dynamically. This refers to the attributes such as the position (top or bottom) of the DIV layer or the text that should be displayed. So we also need a function that can process these attributes and check for changes to these attributes.

Our Javascript code should now look something like this. I've added additional comments to my code so that you'll know later what all these functions mean and what they're good for. Oh yes, the following code is your template for future web components. This code contains all the basic functionality for a web component.

info-banner.js
copied
// Define the InfoBanner-Class.
// This extends HTMLElement, which is the base class for all HTML elements.
class InfoBanner extends HTMLElement {

   // The constructor method is creating and initializing an object created with a class.
   constructor() {
      // Call the constructor of the parent class (HTMLElement).
      super();
      // Attach a shadow root to the custom element. The 'open' mode means that the
      // shadow root can be accessed from Javascript outside the shadow root.
      this.attachShadow({ mode: 'open' });
   }

   // The connectedCallback() lifecycle hook fires
   // when the element gets inserted into the DOM.
   connectedCallback() {
      // Call the RenderComponent method when the element is inserted into the DOM.
      this.RenderComponent();
   }

   // The RenderComponent method will generate
   // the HTML content of the custom element.
   RenderComponent() {
   }

   // The UpdateComponent method will update
   // the HTML content of the custom element.
   UpdateComponent() {
   }

   // The static get observedAttributes method returns an array
   // of attribute names to monitor for changes.
   static get observedAttributes() {
   }

   // The attributeChangedCallback() lifecycle hook fires
   // when an attribute is added, changed, or removed.
   attributeChangedCallback() {
   }
}
//Define the new custom element. This line will trigger the constructor in the InfoBanner class.
//The first argument is the tag name of the new element, and the second argument is the class that controls its behavior.
window.customElements.define('info-banner', InfoBanner);

At this point we need to plan what our component should be able to do. So we need to determine which attributes our component should have. These attributes ultimately determine how dynamic our component becomes. In our example, the component should have the following attributes:

text: Sets the text that will be displayed in our banner
position: Positions the banner either at the top or bottom
bgcolor: Defines the background color of the banner
fontcolor: Defines the text color

In our code we need to define the attributes we want to interact with. Our code already contains the following function: static get observedAttributes()
Within this function we now define our attributes. The function should now look like this:

info-banner.js
copied
static get observedAttributes() {
   // the list of observed attributes
   return ['text', 'position', 'bgcolor', 'fontcolor'];
}

Now that we have set the attributes that we will dynamically change, we can now start implementing the HTML code for our component. We have to do this within the RenderComponent() function.

One thing is important to know at this point:
So that we can process the values of the attributes when creating the component, we need to integrate our attributes into the RenderComponent() and/or UpdateComponent() function.

We will initially only use the text attribute in the RenderComponent() function. The reason is quite simple. When we create the component, we also have to include our external CSS file. To dynamically manipulate the style properties of our component, we first need to inject our CSS code into the Shadow DOM. And that's why we will process all other attributes (like position, bgcolor and fontcolor) in the UpdateComponent() function.

Your RenderComponent() function should look like this now:

info-banner.js
copied
RenderComponent() {
   // Get the attributes. We're going to set some default values
   // as a fallback (if there is an empty value for an attribute)
   const attrText = this.getAttribute('text') || '';

   // Set the inner HTML of the shadow root
   this.shadowRoot.innerHTML = `
   <style>
      @import 'info-banner.css';
   </style>
   <div class="BannerDesign Roboto400">
      <p>${attrText}</p>
   </div>
   `;
   // Call the update method to update other details
   this.UpdateComponent();
}

In the RenderComponent() function we have injected our external info-banner.css using @import;. The line this.shadowRoot.innerHTML renders our component in the Shadow DOM (including the injected CSS code). We determine the text of our banner with this.getAttribute('text') || ''". We then write the text into our component with ${attrText}.

Right at the end of the RenderComponent() function, we internally call our UpdateComponent() function. We have to do this so that we can now process the other attributes.

Here are a few words for the beginners and newbies among us:

It is important to understand that in our case, we need to distinguish between attributes whether they provide content for our component or whether they control the appearance/behavior of our component. We can always process attributes that provide content directly in the RenderComponent() function. But we always need to process attributes that change the appearance or behavior of our component in the UpdateComponent() function. These attributes require that the CSS properties of our component exist in the Shadow DOM so that we can manipulate them. However, the CSS properties of our component are only available when we create the component itself using RenderComponent(). And that's why we can only call the UpdateComponent() function at the very end of our RenderComponent() function.

So now let's move on to our UpdateComponent() function. Here we now need to retrieve the remaining attributes and process their values/contents and apply them to our component.

Your UpdateComponent() function should look like this now:

info-banner.js
copied
UpdateComponent() {
   // Get the attributes. We're going to set some default values
   // as a fallback (if there is an empty value for an attribute)
   const attrPos = this.getAttribute('position') || 'top';
   const attrBcol= this.getAttribute('bgcolor') || '#BEBEBE';
   const attrFcol= this.getAttribute('fontcolor') || '#EFEFEF';

   // Get the info-banner inside the Shadow DOM (by its css class)
   const vBanner = this.shadowRoot.querySelector('.BannerDesign');

   // Check that we have a valid hex-value (if no default value)
   if (/^#[0-9A-F]{6}$/i.test(attrBcol)) {
      // Apply the new background-color to the component
      vBanner.style.backgroundColor = attrBcol;
   }

   // Same check for the font-color (if no default value)
   if (/^#[0-9A-F]{6}$/i.test(attrFcol)) {
      // Apply the new background-color to the component
      vBanner.style.color = attrFcol;
   }

   // Let's set the position of the banner
   if (attrPos === 'top' || attrPos === 'bottom') {
      // Set the position of the banner
      vBanner.style.position = 'fixed';
      // Set the top or bottom property to 16px
      // so we have a distance to the top/bottom
      vBanner.style[attrPos] = '16px';
   }
}

Okay, we're done. That means, stop! We forgot one more thing. We want our component to respond to changes in its attributes (one or more attributes are changed, or attributes are added or removed). For this we have already defined the attributeChangedCallback() function in our Javascript. The attributeChangedCallback() function is called every time changes are made to the attributes that we defined with static get observedAttributes(). At this point we can make it easy for ourselves. When changes occur, we simply re-render our entire component.

Your attributeChangedCallback() function should look like this now:

info-banner.js
copied
attributeChangedCallback() {
   // Call the RenderComponent function to re-render the component
   this.RenderComponent();
}

Congratulations. You have now created your first web component. Because we are now finished with the official part of the programming. Our CSS code is complete, the Javascript file contains the complete code to create our web component. We have implemented all the necessary functions to dynamically change the desired attributes. All we need now is an HTML file that integrates our web component and... if everything is correct and works as planned... displays it correctly.

So let's edit our info-banner.html and integrate our web component there. When we open the HTML file, we first remove all the code between the body tags. And since our web component comes with its own CSS, we can also remove the reference to the external stylesheet in the header. Our web component has its own HTML tag (<info-banner></info-banner>). We also defined four attributes (text, position, bgcolor and fontcolor). So we just use our web component just like any other HTML tag.

And this is now the code of your HTML file:

info-banner.html
copied
<!DOCTYPE html>
<html lang="de">
<head>
   <meta charset="UTF-8">
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   <title>Web-Component: info-banner</title>
   <script src="info-banner.js"></script>
</head>
<body>
   <info-banner
      text="Hello, Web Components!"
      position="top"
      bgcolor="#0088DA"
      fontcolor="#EFEFEF">
   </info-banner>
</body>
</html>
